# 🔐 คู่มือการติดตั้งและจัดทำระบบฟอร์มลงทะเบียน SSL-VPN
ระบบนี้เป็นเว็บฟอร์มลงทะเบียนเชื่อมต่อระยะไกลแบบปลอดภัย (SSL-VPN Registration Form) แบ่งการกรอกข้อมูลออกเป็น 3 ขั้นตอนหลัก (Multi-step Form) ควบคุมการทำงานของ **Frontend (หน้าบ้าน)** ด้วย HTML, CSS, JavaScript และประมวลผลข้อมูลส่งอีเมลยืนยันตัวตนด้วย OTP รวมถึงบันทึกข้อมูลอย่างปลอดภัยในฝั่ง **Backend (หลังบ้าน)** ด้วย **Google Apps Script** และ **Google Sheets**

---

## 📋 สิ่งที่ต้องเตรียมความพร้อมก่อนเริ่มต้น (Prerequisites)
หากต้องการทำเว็บระบบลงทะเบียนนี้ให้ใช้งานได้จริง คุณจำเป็นต้องเตรียมสิ่งเหล่านี้:

1. **บัญชี Google (Google Account)**: สำหรับใช้งาน Google Drive, Google Sheets และ Gmail
2. **Google Spreadsheet (ทำหน้าที่เป็นฐานข้อมูล)**: เพื่อเก็บรายชื่อผู้ลงทะเบียนและการออกรหัส OTP
3. **Google Apps Script Web App**: เพื่อทำหน้าที่เป็น API Backend คอยรับคำสั่งส่ง OTP, ยืนยันรหัส OTP และบันทึกข้อมูลแบบ Masking ป้องกัน PDPA
4. **ไฟล์หน้าเว็บส่วนของระบบ (Frontend)**:
   * **Step 1 (ข้อมูลเบื้องต้น & ขอ OTP)**: `index.html`, `style.css`, `script.js`
   * **Step 2 (นโยบายความคุ้มครองข้อมูลส่วนบุคคล - PDPA)**: `page2/index.html`, `page2/page2.css`, `page2/page2.js`
   * **Step 3 (เงื่อนไขการใช้งานเทคโนโลยีสารสนเทศ - AUP & ส่งข้อมูล)**: `page3/index.html`, `page3/page3.css`, `page3/page3.js`
5. **พื้นที่จัดวางหน้าเว็บ (Web Hosting)**: บริการเซิร์ฟเวอร์สำหรับเผยแพร่เว็บไซต์ เช่น GitHub Pages, Netlify, Vercel หรือเซิร์ฟเวอร์ส่วนตัว **(ต้องเป็น HTTPS เท่านั้น)**

---

## 🛠️ 4 ขั้นตอนในการติดตั้งระบบทั้งหมด (Step-by-Step Setup Guide)

---

### 1️⃣ ขั้นตอนที่ 1: การเตรียมฐานข้อมูลบน Google Sheets
1. เข้าไปที่ [Google Drive](https://drive.google.com) ของคุณ
2. คลิกปุ่ม **+ ใหม่ (+ New)** -> เลือก **Google ชีต (Google Sheets)** เพื่อสร้างไฟล์เอกสารแผ่นงานใหม่
3. ตั้งชื่อไฟล์ตามความต้องการ เช่น `SSL-VPN Registration Database`
4. ทำการ**คัดลอก Spreadsheet ID** จาก URL บนเบราว์เซอร์ของคุณ
   * *ตัวอย่าง URL:* `https://docs.google.com/spreadsheets/d/1TYEHQBspkpfneiTBwBFw7xR5AaZ5yktDIKlIIR62mCM/edit#gid=0`
   * *Spreadsheet ID คือรหัสช่วงนี้:* `1TYEHQBspkpfneiTBwBFw7xR5AaZ5yktDIKlIIR62mCM` (ให้เก็บรหัสนี้ไว้สำหรับขั้นตอนถัดไป)
5. **ข้อแนะนำพิเศษ**: ไม่ต้องทำการกรอกชื่อคอลัมน์หรือสร้างหัวตารางใดๆ ทิ้งไว้แบบว่างๆ ได้เลย เนื่องจาก Backend ของเราได้รับการเขียนให้เตรียมสร้างและตกแต่งหัวตารางของชีต `OTP_Log` และ `Registrations` ให้เรียบร้อยแบบอัตโนมัติในครั้งแรกที่ฟอร์มทำงาน

---

### 2️⃣ ขั้นตอนที่ 2: ติดตั้งระบบควบคุมข้อมูล Google Apps Script (Backend)
1. ในหน้าเอกสาร Google Sheets ที่คุณเพิ่งสร้างขึ้น ให้มองหาแถบเมนูด้านบน
2. คลิกที่เมนู **ส่วนขยาย (Extensions)** -> เลือก **Apps Script**
3. หน้าต่างโปรแกรมสำหรับเขียนสคริปต์จะเปิดขึ้นมาใหม่ คุณจะพบไฟล์ชื่อ `รหัส.gs` (หรือ `Code.gs`)
4. ให้ล้างข้อความเดิมในไฟล์ออกทั้งหมด แล้วทำการ**คัดลอกโค้ดจากไฟล์ `Code.gs`** ในโปรเจกต์ของคุณมาวางลงไปทั้งหมด
5. ค้นหาแถวที่มีตัวแปรกำหนด ID สเปรดชีต (บรรทัดที่ 25) แล้วแก้ไขให้เป็น ID ของคุณ:
   ```javascript
   const SPREADSHEET_ID = 'ใส่_SPREADSHEET_ID_ของคุณที่นี่';
   ```
   *นำรหัส Spreadsheet ID ที่คัดลอกไว้ในขั้นตอนที่ 1 มาวางแทนในเครื่องหมายคำพูด*
6. กดปุ่ม **บันทึกโครงการ (Save project)** รูปแผ่นดิสก์บริเวณแถบเครื่องมือด้านบน

---

### 3️⃣ ขั้นตอนที่ 3: เปิดบริการ Apps Script เป็น Web App (Deployment)
การทำให้ระบบหน้าเว็บและฐานข้อมูลหลังบ้านพูดคุยติดต่อกันได้ คุณต้องทำตามขั้นตอนนี้เพื่อนำ URL ตัวกลางมาใช้:
1. ด้านบนขวาของหน้าจอ Apps Script คลิกปุ่ม **ใช้งานได้จริง (Deploy)** -> เลือก **การจัดการการจัดวางใหม่ (New deployment)**
2. ในหน้าต่างป๊อปอัป ให้คลิกปุ่มรูปฟันเฟืองบริเวณหัวข้อเลือกประเภทของการ Deploy -> เลือก **เว็บแอป (Web app)**
3. ตั้งค่าการกำหนดค่า (Configuration) ของเว็บแอปดังนี้:
   * **คำอธิบาย (Description)**: ตั้งตามใจชอบ (เช่น `SSL-VPN Form Backend v1`)
   * **เรียกใช้งานในฐานะ (Execute as)**: เลือก **ฉัน (Me - บัญชีอีเมลเจ้าของสเปรดชีต)**
   * **ผู้มีสิทธิ์เข้าถึง (Who has access)**: เลือก **ทุกคน (Anyone)** *(ส่วนนี้ห้ามเลือกผิดเด็ดขาด เพื่อให้หน้าเว็บส่งคำขอมาได้โดยไม่ต้องล็อกอินบัญชีกูเกิล)*
4. คลิกปุ่ม **จัดวางใช้งานได้จริง (Deploy)**
5. หน้าจอจะขอสิทธิ์ควบคุมพื้นที่ ให้คลิก **ให้สิทธิ์เข้าถึง (Authorize access)** -> เลือกบัญชีอีเมล Google ของคุณ
6. ในกรณีที่พบหน้าต่างแจ้งเตือนความปลอดภัยว่า Google ยังไม่ได้ตรวจสอบสิทธิ์แอปนี้ (Google hasn't verified this app):
   * คลิกที่หัวข้อ **ขั้นสูง (Advanced)** มุมซ้ายล่าง
   * คลิกเลือกลิงก์ **ไปที่... (Go to... (unsafe))** ด้านล่างสุด
   * คลิกปุ่ม **อนุญาต (Allow)** เพื่อยินยอมสิทธิ์
7. เมื่อเสร็จสมบูรณ์ จะปรากฏหน้าจอแสดงผลลัพธ์ ให้มองหาหัวข้อ **เว็บแอป (Web app)**
8. ทำการ**คัดลอก URL ของเว็บแอป** ที่ได้รับมา (มีลักษณะ URL คล้ายแบบนี้: `https://script.google.com/macros/s/AKfycb.../exec`)

---

### 4️⃣ ขั้นตอนที่ 4: ทำการตั้งค่าจุดเชื่อมโยงหน้าเว็บ Frontend
1. ย้อนกลับมาเปิดโฟลเดอร์งานเว็บไซต์ในเครื่องคอมพิวเตอร์ของคุณ
2. ค้นหาและเปิดไฟล์ **`script.js`** (ไฟล์จัดการหน้าลงทะเบียน Step 1)
   * ไปที่บรรทัดที่ 23 สังเกตตัวแปร `GAS_URL`:
     ```javascript
     const GAS_URL = 'https://script.google.com/macros/s/.../exec';
     ```
   * วาง Web App URL ที่เพิ่งคัดลอกมาแทนที่ URL เดิมลงไปในเครื่องหมายคำพูดเดี่ยว
3. ค้นหาและเปิดไฟล์ **`page3/page3.js`** (ไฟล์จัดการหน้าลงทะเบียน Step 3)
   * ไปที่บรรทัดที่ 16 สังเกตตัวแปร `GAS_URL`:
     ```javascript
     const GAS_URL = 'https://script.google.com/macros/s/.../exec';
     ```
   * ทำการแก้ไขวาง Web App URL ตัวเดียวกันนี้ทดแทนค่าเดิม
4. ทำการกดบันทึก (Save) ไฟล์งาน JavaScript ทั้งสองจุดให้เรียบร้อย

---

## 🌐 การทดสอบความถูกต้องและการเผยแพร่ระบบ (Testing & Hosting)

### 🧪 ขั้นตอนการทดสอบการทำงานระบบ (Local Testing)
1. เปิดไฟล์ `index.html` บนโฟลเดอร์หลักผ่านเว็บเบราว์เซอร์บนเครื่องคอมพิวเตอร์
2. ทดลองกรอกฟอร์มใน Step 1 ให้ครบทุกช่อง (หากข้อมูลใดมีรูปแบบไม่ถูกต้อง เช่น บัตรประชาชนไม่ครบ 13 หลัก หรืออีเมลไม่ถูกต้อง ปุ่มขอรหัสจะถูกปิดการทำงานอยู่)
3. กดปุ่ม **Request OTP** -> ตรวจสอบในกล่องจดหมายของคุณ คุณจะได้รับรหัสตัวเลข 6 หลักผ่านทางอีเมล (ส่งออกผ่าน MailApp ใน Gmail)
4. ป้อนตัวเลข 6 หลักในช่อง OTP -> เมื่อป้อนครบระบบหน้าเว็บจะทำการยืนยันตัวตนกับฐานข้อมูลทันที เมื่อผ่านจะเข้าสู่สถานะสีเขียว (Verified)
5. คลิก **Next** เพื่อเข้าสู่ Step 2 -> เลื่อนอ่านเงื่อนไขความปลอดภัยและกดยอมรับนโยบาย
6. คลิก **Next** เพื่อเข้าสู่ Step 3 -> เลื่อนอ่านนโยบายการใช้งานไอที (AUP) และกดยืนยันแล้วกด **Submit**
7. ตรวจสอบเมื่อระบบทำงานเสร็จสิ้น ป๊อปอัปความสำเร็จจะขึ้นแสดงข้อมูลสรุปที่บันทึก
8. กลับไปเปิดหน้า Google Sheets ของคุณ จะพบข้อมูลส่วนบุคคลถูกส่งไปบันทึกอย่างสมบูรณ์แบบ ทั้งนี้ข้อมูลบางส่วน เช่น เลขบัตรประชาชน, เบอร์โทร, อีเมล จะถูกเซนเซอร์ปกปิด (Masked) เพื่อความปลอดภัยตามมาตรฐาน PDPA เรียบร้อยอย่างสวยงาม!

### 🚀 การอัปโหลดหน้าเว็บขึ้นอินเทอร์เน็ต (Web Deployment)
เมื่อทดสอบทุกอย่างเรียบร้อยดีแล้ว คุณสามารถนำไฟล์และโฟลเดอร์ทั้งหมดไปอัปโหลดขึ้นโฮสติ้งเพื่อเปิดใช้งานเป็นเว็บจริง เช่น:
* อัปโหลดไฟล์โปรเจกต์ขึ้นสู่ **GitHub** และตั้งค่าการเปิดใช้ **GitHub Pages**
* หรือการลากวางโปรเจกต์เพื่อนำขึ้นใช้งานฟรีแบบง่ายๆ ผ่านทางบริการของ **Netlify** หรือ **Vercel**
